mysql的数据结构

# 存储引擎

  • InnoDB MySQL默认的存储引擎，支持事务，行级锁，外键等特性
  • MyISAM 支持表级别锁，不支持事务，数据文件和索引文件是分开存储
  • Memory 将数据存储在内存中，速度快，但是不能持久化

# 索引数据结构

# 1. BTree索引

数据库默认的索引结构，支持范围查找 ，查询速度稳定

  • 数据保存在叶子节点，每次访问的层级都是一样的，所以查询速度稳定
  • 叶子节点双向链表关联，支持范围查找
  • 非主键索引的叶子节点是保存的是主键的ID，查询的时候要通过回表获取行数据
